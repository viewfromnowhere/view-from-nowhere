Philosophy — evidence before interpretation

Objectivity is not freedom from perspective; it is the discipline of exposing it. — after Thomas Nagel, The View from Nowhere
  • Transparency over persuasion. Every transformation leaves a footprint.
  • Provenance before narrative. Sources and links are first-class.
  • Reproducibility over authority. Same inputs → same outputs.
  • Tools, not takes. Infrastructure that lets the public audit how we know.

Nowhere is not a newsroom. It is the substrate on which rigorous journalism can be practiced in the open.

For journalists — a claim-centric workflow

  1. Declare a claim. Precisely state the proposition under review.
  2. Gather artifacts. Search social, documents, archives; capture context.
  3. Normalize & link. Extract entities/claims; attach typed provenance (supports/refutes/derived-from).
  4. Review. Human-in-the-loop acceptance, annotation, and counter-evidence.
  5. Publish. Export an auditable bundle: evidence graph + logs + config.

Designed for independent reporters, watchdogs, civic data orgs, and newsrooms with technical staff.

For engineers — how it works

  • Rust actor runtime orchestrates acquisition → normalization → storage with bounded mailboxes and shared rate limiting.
  • LLM adapters (OpenAI, Ollama) power search and normalization with prompt_version recorded for determinism.
  • Evidence graph (v1) on SQLite: record, link, claim, FTS views, append-only semantics.
  • TUI control plane: set claims, watch pipelines, accept/reject artifacts, export signed bundles.
  • Modular crates: config, HTTP, social, drivers, observability — clean boundaries, swappable integrations.

Roadmap — toward evidence-first journalism

  • Evidence Graph v1 → v2: typed records/claims/links; deterministic views; signed commits.
  • Evidence-trained models: fine-tune LLMs on graph-grounded corpora to favor provenance over speculation.
  • Reproducible pipelines: locked models/prompts; replayable investigations.
  • Distributed event store: append-only DAG with peer-verifiable bundles.
  • Offline/field mode: secure capture; later sync; reproducible builds.

See VISION.md and ARCHITECTURE.md in the repo for the long arc.

Get started

  1. Clone the repo and copy the environment template: 
    $ git clone <repo-url>
$ cd view-from-nowhere
$ cp .env.example .env
  2. Set secrets in .env: DATABASE_URL, OPENAI_API_KEY, TWITTER_BEARER_TOKEN.
  3. Initialize the database: 
    $ sqlite3 nowhere.db < migrations/01_init.sql
  4. Review nowhere.yaml, toggle actors and models.
  5. Run the TUI: 
    $ cargo run -p nowhere-app
